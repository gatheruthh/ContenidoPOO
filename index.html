De la Estructura a la Abstracción

Una guía práctica basada en el modelado UML y la implementación de nómina para entender el paradigma orientado a objetos.

Objetivo del Aprendizaje

Según la Unidad I y II de nuestra guía temática, el estudiante aprenderá a:

  • Identificar la diferencia entre funciones y objetos.
  • Aplicar Encapsulamiento, Herencia y Polimorfismo.
  • Modelar soluciones mediante diagramas UML.

Competencia a Desarrollar:

"El estudiante representará clases en lenguaje de programación orientado a objetos para codificar el análisis y diseño de modelos en UML."

Progresión del Paradigma

Paradigma Estructurado

En esta etapa, el código depende de variables sueltas en el Main y funciones estáticas. La lógica de datos y comportamiento está separada.

// Ejemplo de lógica estructurada
static float CalcularNominaAdmon(float sueldoBase, float isr) {
    return sueldoBase - (sueldoBase * isr / 100);
}
Debilidad: Difícil de escalar. Si agregas más tipos de empleados, el switch del Main crecerá infinitamente.

Introducción a Clases

Aquí empezamos a agrupar el comportamiento. Creamos clases para representar a los empleados, pero aún pasamos parámetros a los métodos.

internal class clsAdministrativos {
    public float CalcularNominaAdmon(float sueldoBase, float isr) {
        return sueldoBase - (sueldoBase * isr / 100);
    }
}

Clases con Estado (Atributos)

¡Abstracción real! El objeto ahora "sabe" quién es y cuáles son sus valores. El método ya no necesita recibir parámetros porque lee sus propios atributos.

internal class clsAdministrativos {
    public string nombreTrabajador;
    public float sueldoBase;
    public float isr;

    public float CalcularNominaAdmon() {
        return sueldoBase - (sueldoBase * isr / 100);
    }
}

Herencia, Abstracción y Polimorfismo

Aplicamos el Modelo UML. Identificamos que todos son "Trabajadores". Usamos una abstract class para definir la estructura común.

  • Herencia: clsAdministrativos : clsTrabajadores
  • Encapsulamiento: Uso de private y Properties.
  • Polimorfismo: override float CalcularNomina()
Diagrama de Clases (Abstracto)
[clsTrabajadores]
▲
├── [clsAdministrativos]
├── [clsProfesorAsignatura]
└── [clsProfesorTiempoCompleto] 
internal abstract class clsTrabajadores {
    public string nombreTrabajador;
    protected float sueldoNeto;
    public abstract float CalcularNomina(); // Polimorfismo puro
}
